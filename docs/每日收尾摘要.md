## 2026-05-23

### 本次改动概述

- 增加状态版本号和写入冲突保护：前端保存数据时会带上当前 `stateRevision`，后端发现旧页面覆盖新数据时返回 409，并把最新数据同步回前端。
- 增强持久化合并逻辑：保存项目、公共提醒和推送记录时，保留提醒发送状态、确认状态、回执状态和推送历史，降低旧页面把线上提醒状态覆盖掉的风险。
- API 响应增加 `no-store` 缓存控制，避免登录、状态接口被浏览器或中间层缓存。
- 前端在页面重新获得焦点、从后台切回或浏览器恢复页面时自动刷新远端状态，并在部署版本变化时自动刷新页面。
- 产品工艺卡的复选框字段支持多个可选项，保存时会校验必填项并记录多选结果。
- 新增共享用户名规则模块，统一姓名转拼音账号、默认初始密码和用户名预览逻辑。
