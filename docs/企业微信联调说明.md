# 企业微信联调说明

这版已经把企业微信的第一层接入骨架放进项目里了，目标不是一次做完所有智能能力，而是先把“能接、能记、能回、能发”这条链路打通。

## 当前已接上的接口

1. `/api/wecom/callback`
   用来接企业微信回调、校验签名、解密消息、写入 WorkPad 状态，并返回被动回复。

2. `/api/wecom/send-message`
   用来主动给企业微信成员发应用消息。

3. `/api/wecom/health`
   用来检查企微配置、消息收件箱数量，以及当前存储模式。

4. `/api/cron/dispatch-reminders`
   用来按分钟扫描待提醒项目，并通过企业微信主动推送到提醒人。

## 当前支持的消息指令

1. `帮助`
2. `查看 BK-2026-001`
3. `记录 BK-2026-001 作者已回稿，准备二校`
4. `提醒 陈敏 BK-2026-005 2026-05-20 15:30 催合同回签`
5. `我的提醒`
6. `绑定 贾涛`
7. 语音或自然语言：`明天下午三点提醒张莹跟进 BK-2026-005 合同回签`

## 你需要准备的环境变量

### 回调模式

1. `WECOM_CORP_ID`
2. `WECOM_TOKEN`
3. `WECOM_ENCODING_AES_KEY`

### 主动发消息

1. `WECOM_APP_SECRET`
2. `WECOM_AGENT_ID`
3. 如果企业微信自建应用开启了可信 IP，必须把 WorkPad 服务端出口 IP 加入可信 IP / IP 白名单。否则获取 `access_token`、下载语音素材、主动发送消息都会报 `60020 not allow to access from your ip`。
4. Vercel Serverless 出口 IP 可能变化，不建议长期手动追 IP。稳定方案是启用固定 IP 代理。

### 到点提醒派发

1. `CRON_SECRET`：Vercel Cron 调用 `/api/cron/dispatch-reminders` 时使用的密钥。
2. `vercel.json` 已配置每分钟扫描一次待提醒项目。
3. 只有通过企业微信或后台明确新建/修改过的提醒会进入待推送队列，避免演示数据里的旧提醒集中推送。
4. 如果企业微信可信 IP 还没稳定，派发会记录失败并稍后重试；启用固定 IP 代理后会更稳定。

### 固定 IP 代理

1. `WECOM_PROXY_BASE_URL`：固定 IP 代理的 HTTPS 地址，例如 `https://wecom-proxy.tbxprint.com`。
2. `WECOM_PROXY_SECRET`：WorkPad 调用代理时使用的共享密钥，必须和代理服务器上的 `WORKPAD_PROXY_SECRET` 一致。
3. 代理代码在 `services/wecom-fixed-ip-proxy`，适合部署到阿里云 ECS + 固定公网 IP / EIP。
4. 启用后，WorkPad 会把获取 `access_token`、下载语音素材、主动发送应用消息都转到固定 IP 代理上。
5. 企业微信后台“企业可信 IP / IP 白名单”只需要填写代理服务器的固定公网 IP，不再填写 Vercel 每次变化的临时出口 IP。

### 线上持久化

1. `BLOB_READ_WRITE_TOKEN`

如果不配 `BLOB_READ_WRITE_TOKEN`，当前线上虽然能跑接口，但会退化成临时缓存，不适合正式生产。

### 语音与自然语言解析

1. `AI_API_KEY`：默认共用云雾 Key。
2. `AI_COMMAND_MODEL`：可选，默认 `deepseek-v4-flash`，用于把自然语言解析成提醒命令。
3. `AI_TRANSCRIPTION_MODEL`：可选，默认 `whisper-1`，用于语音转文字。
4. 如果企业微信回调里已经带 `Recognition` 识别文本，系统会优先使用它；否则会通过 `MediaId` 下载语音临时素材，再调用语音转文字模型。
5. 如果下载语音时报 `60020 not allow to access from your ip`，说明企业微信没有放行当前服务器出口 IP。可以先按错误提示里的 `from ip` 加白名单；长期稳定方案是启用 `services/wecom-fixed-ip-proxy` 固定 IP 代理。

## 你在企业微信后台需要做的事

1. 创建或确认一个自建应用。
2. 记下 `CorpID`、`AgentId`、应用 `Secret`。
3. 在接收消息配置里填写：
   - URL：`https://workpad.tbxprint.com/api/wecom/callback`
   - Token：填到 `WECOM_TOKEN`
   - EncodingAESKey：填到 `WECOM_ENCODING_AES_KEY`
4. 把会用到的员工加入这个应用可见范围。
5. 按照 WorkPad 后台里的“人员录入”，把每个人的 `企微账号 UserId` 补齐。
6. 如果已经启用固定 IP 代理，只把代理 `/health` 返回的 `egressIp` 填入企业微信可信 IP。

## 推荐联调顺序

1. 先打开 `/api/wecom/health`，确认配置状态。
2. 再去企业微信后台做 URL 验证。
3. 验证通过后，先发 `帮助` 和 `查看 BK-2026-001`。
4. 如果当前企微账号还没映射，先发 `绑定 贾涛` 这类命令完成自助绑定。
5. 再测 `记录` 和 `提醒`。
6. 发一条语音，例如“明天下午三点提醒张莹跟进 BK-2026-005 合同回签”。
7. 最后再接主动消息发送。
8. 如果启用固定 IP 代理，打开 `/api/wecom/health`，确认 `fixedIpProxyConfigured` 是 `true`。
9. 设置一条未来几分钟的提醒，等到点后确认提醒人能收到企业微信主动消息。

## 当前边界

1. 语音转文字依赖企业微信回调识别文本或后台配置的语音转文字模型；如果模型不支持企业微信语音格式，需要在后台切换可用 ASR 模型。
2. 暂时还没有把企微消息收件箱做成后台页面，只是已经入库。
3. 主动消息发送已经有接口，但还没挂到前台按钮。
4. 固定 IP 代理需要单独部署到有固定公网 IP 的服务器；仅把代码放到 Vercel 不能解决企业微信可信 IP 变化问题。
